## 2024-07-01 - [Large File ArrayBuffer to Base64 Conversion]
**Learning:** Using a single byte-by-byte `for` loop with `binary += String.fromCharCode(bytes[i])` for large files (like GCP Service Account JSONs or other credentials up to ~5MB) in `FileCredForm.tsx` is an `O(n^2)` operation due to repeated string re-allocation in V8, causing noticeable browser freezes/latency during file upload. On the other hand, using spread syntax `String.fromCharCode(...bytes)` causes "Maximum call stack size exceeded".
**Action:** Always process large `Uint8Array` to string conversions using a chunked approach with `Array.from(chunk)` and `String.fromCharCode.apply(null, Array.from(chunk))` (or `chunk as unknown as number[]` depending on TypeScript config), ideally with a chunk size around 8192, to avoid stack overflow while retaining near `O(n)` string construction performance.
