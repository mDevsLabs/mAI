## 2024-07-01 - Uint8Array to Base64 Optimization
**Learning:** Using `reduce` (e.g., `.reduce((data, byte) => data + String.fromCharCode(byte), '')`) or full array spread (`String.fromCharCode(...data)`) to convert large `Uint8Array`s to Base64 strings is incredibly inefficient in V8/Node/Browsers. It causes severe performance degradation, memory allocation overhead, and throws `Maximum call stack size exceeded` errors for arrays larger than ~100KB due to JavaScript engine argument limits. Our benchmark showed a 5MB array taking ~1.62s with `reduce` compared to ~224ms with chunking.
**Action:** Always use chunked processing (e.g., `String.fromCharCode.apply(null, Array.from(bytes.subarray(i, i + 8192)))`) for Base64 conversion of binary data when a native implementation like `Buffer.from(buf).toString('base64')` is unavailable (such as in client-side code).
