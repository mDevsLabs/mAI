## ✨ Funktionen

Heutige Agenten sind einmalige, aufgabenorientierte Werkzeuge. Es fehlt ihnen an Kontext, sie leben isoliert und erfordern manuelle Übergaben zwischen verschiedenen Fenstern und Modellen. Selbst wenn einige ein Gedächtnis besitzen, ist dieses oft global, oberflächlich und unpersönlich. In diesem Modus sind die Nutzer gezwungen, zwischen fragmentierten Konversationen hin- und herzuschalten, was eine strukturierte Produktivität erschwert.

**mAI ändert alles.**

mAI ist ein Arbeits- und Lebensraum, um Agenten-Teampartner zu finden, aufzubauen und mit ihnen zusammenzuarbeiten, die mit Ihnen wachsen. In mAI betrachten wir **Agenten als Arbeitseinheit** und bieten eine Infrastruktur, in der sich Menschen und Agenten gemeinsam weiterentwickeln.

### Operator: Agenten als Arbeitseinheit

Stellt Ihr gesamtes KI-Team ein, plant deren Einsätze und erstattet Bericht.

- **Mehr Produktivität. Weniger Tools**: Bringen Sie all Ihre Agenten unter ein Dach.
- **IM-Gateway**: Agenten direkt dort, wo Sie bereits chatten.

### Erstellen: Agenten als Arbeitseinheit

Der Aufbau eines personalisierten KI-Teams beginnt mit dem **Agent Builder**. Sie können einmal beschreiben, was Sie benötigen, und die Einrichtung des Agenten startet sofort mit automatischen Konfigurationen, sodass Sie ihn direkt verwenden können.

- **Einheitliche Intelligenz**: Greifen Sie nahtlos auf jedes Modell und jede Modalität zu — alles unter Ihrer Kontrolle.
- **Über 10.000 Skills**: Verbinden Sie Ihre Agenten mit den Fähigkeiten, die Sie täglich nutzen, dank einer Bibliothek mit über 10.000 Tools und MCP-kompatiblen Plugins.

### Kollaborieren: Neue Formen von Kollaborationsnetzwerken skalieren

mAI führt **Agentengruppen** ein, sodass Sie mit Agenten wie mit echten Teammitgliedern zusammenarbeiten können. Das System stellt die passenden Agenten für die jeweilige Aufgabe zusammen, was eine parallele Zusammenarbeit und iterative Verbesserungen ermöglicht.

- **Pages (Seiten)**: Schreiben und verfeinern Sie Inhalte mit mehreren Agenten an einem Ort und mit gemeinsamem Kontext.
- **Zeitplan**: Planen Sie Ausführungen und lassen Sie die Agenten die Arbeit zur richtigen Zeit erledigen, selbst wenn Sie abwesend sind.
- **Projekte**: Organisieren Sie Ihre Arbeit nach Projekten, um alles strukturiert und leicht nachverfolgbar zu halten.
- **Workspace (Arbeitsbereich)**: Ein gemeinsamer Raum für Teams zur Zusammenarbeit mit Agenten, der eine klare Zuweisung und Sichtbarkeit im gesamten Unternehmen gewährleistet.

### Weiterentwickeln: Co-Evolution von Menschen und Agenten

Die beste KI ist diejenige, die Sie tiefgehend versteht. mAI bietet ein **persönliches Gedächtnis** (Personal Memory), das ein klares Verständnis für Ihre Bedürfnisse aufbaut.

- **Kontinuierliches Lernen**: Ihre Agenten lernen aus Ihrer Arbeitsweise und passen ihr Verhalten an, um im richtigen Moment zu agieren.
- **White-Box-Gedächtnis**: Wir glauben an Transparenz. Ihre Agenten nutzen ein strukturiertes, bearbeitbares Gedächtnis, sodass Sie die volle Kontrolle darüber haben, woran sie sich erinnern.

## 🛳 Self-Hosting

mAI bietet eine selbstgehostete Version für Vercel, Alibaba Cloud und als [Docker-Image][docker-release-link]. Dies ermöglicht es Ihnen, Ihren eigenen Chatbot in wenigen Minuten und ohne Vorkenntnisse bereitzustellen.

> [!TIP]
>
> Erfahren Sie mehr über das [📘 Hosten Ihres eigenen mAI][docs-self-hosting].

### `A` Bereitstellung mit Vercel, Zeabur, Sealos oder Alibaba Cloud

Wenn Sie diesen Dienst selbst auf Vercel, Zeabur oder Alibaba Cloud bereitstellen möchten, folgen Sie diesen Schritten:

- Bereiten Sie Ihren [OpenAI-API-Schlüssel](https://platform.openai.com/account/api-keys) vor.
- Klicken Sie auf die Schaltfläche unten, um die Bereitstellung zu starten: Melden Sie sich direkt mit Ihrem GitHub-Konto an und denken Sie daran, den `OPENAI_API_KEY` (erforderlich) im Bereich der Umgebungsvariablen einzutragen.
- Nach der Bereitstellung können Sie mit der Nutzung beginnen.
- Eigene Domain einbinden (optional): Da DNS-Einträge der von Vercel zugewiesenen Domains in einigen Regionen blockiert sind, kann die Anbindung einer eigenen Domain eine direkte Verbindung ermöglichen.

Nach dem Forken sollten Sie nur die Upstream-Synchronisierungsaktion (upstream sync) beibehalten und alle anderen GitHub-Actions in Ihrem Repository deaktivieren.

Wenn Sie Ihr eigenes Projekt mithilfe der Ein-Klick-Bereitstellung im README bereitgestellt haben, erhalten Sie möglicherweise ständig die Meldung „Updates verfügbar“. ### `B` Bereitstellung mit Docker

[![][docker-release-shield]][docker-release-link]
[![][docker-size-shield]][docker-size-link]
[![][docker-pulls-shield]][docker-pulls-link]

Wir stellen ein Docker-Image zur Bereitstellung des mAI-Dienstes auf Ihrem eigenen privaten Gerät bereit. Verwenden Sie den folgenden Befehl, um den mAI-Dienst zu starten:

1. Erstellen Sie einen Ordner für die Speicherung der Dateien:

```fish
$ mkdir mAI-db && cd mAI-db
```

2. Initialisieren Sie die mAI-Infrastruktur:

```fish
bash <(curl -fsSL https://lobe.li/setup.sh)
```

3. Starten Sie den mAI-Dienst:

```fish
docker compose up -d
```

> [!NOTE]
>
> Ausführliche Anweisungen zur Bereitstellung mit Docker finden Sie im [📘 Docker-Bereitstellungshandbuch][docs-docker]

### Umgebungsvariablen

Dieses Projekt bietet einige zusätzliche Konfigurationsoptionen, die über Umgebungsvariablen festgelegt werden können:

| Umgebungsvariable | Erforderlich | Beschreibung | Beispiel |
| -------------------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| `OPENAI_API_KEY` | Ja | Dies ist der API-Schlüssel, den Sie auf Ihrer OpenAI-Kontoseite beantragen. | `sk-xxxxxx...xxxxxx` |
| `OPENAI_PROXY_URL` | Nein | Wenn Sie einen Proxy für die OpenAI-Schnittstelle konfigurieren, können Sie diese Option nutzen, um die Standard-Basis-URL der OpenAI-API-Anfragen zu überschreiben. | `https://api.chatanywhere.cn` oder `https://aihubmix.com/v1` <br/>Der Standardwert ist<br/>`https://api.openai.com/v1` |
| `OPENAI_MODEL_LIST` | Nein | Dient zur Steuerung der Modellliste. Verwenden Sie `+`, um ein Modell hinzuzufügen, `-`, um ein Modell auszublenden, und `modell_name=anzeigename`, um den Anzeigenamen eines Modells anzupassen (jeweils durch Kommas getrennt). | `qwen-7b-chat,+glm-6b,-gpt-3.5-turbo` |

> [!NOTE]
>
> Die vollständige Liste der Umgebungsvariablen finden Sie in den [📘 Umgebungsvariablen][docs-env-var]

## 🧩 Plugins

Plugins bieten eine Möglichkeit, die [Function Calling][docs-function-call]-Fähigkeiten von mAI zu erweitern. Sie können verwendet werden, um neue Funktionsaufrufe und sogar neue Methoden zur Darstellung von Nachrichtenergebnissen einzuführen. Wenn Sie an der Plugin-Entwicklung interessiert sind, lesen Sie bitte unser [📘 Plugin-Entwicklungshandbuch][docs-plugin-dev] im Wiki.

- [lobe-chat-plugins][lobe-chat-plugins]: Dies ist das Plugin-Verzeichnis für mAI. Es greift auf die Datei index.json aus diesem Repository zu, um dem Benutzer eine Liste der verfügbaren mAI-Plugins anzuzeigen.
- [chat-plugin-template][chat-plugin-template]: Dies ist die Plugin-Vorlage für die mAI-Plugin-Entwicklung.
- [@lobehub/chat-plugin-sdk][chat-plugin-sdk]: Das mAI Plugin SDK unterstützt Sie bei der Erstellung herausragender Chat-Plugins für mAI.
- [@lobehub/chat-plugins-gateway][chat-plugins-gateway]: Das mAI Plugins Gateway ist ein Backend-Dienst, der ein Gateway für mAI-Plugins bereitstellt. Wir stellen diesen Dienst über Vercel bereit. Die primäre API POST /api/v1/runner is als Edge Function bereitgestellt.

> [!NOTE]
>
> Das Plugin-System befindet sich derzeit in einer umfassenden Entwicklungsphase. Mehr erfahren Sie in den folgenden Issues:
>
> - [x] [**Plugin-Phase 1**](https://github.com/mDevsLabs/mAI/issues/73): Implementierung der Trennung des Plugins vom Hauptprogramm, Aufteilung des Plugins in ein unabhängiges Repository zur Wartung und Realisierung des dynamischen Ladens von Plugins.
> - [x] [**Plugin-Phase 2**](https://github.com/mDevsLabs/mAI/issues/97): Sicherheit und Stabilität der Plugin-Nutzung, präzisere Darstellung von Fehlerzuständen, Wartbarkeit der Plugin-Architektur und Entwicklerfreundlichkeit.
> - [x] [**Plugin-Phase 3**](https://github.com/mDevsLabs/mAI/issues/149): Erweiterte und umfassendere Anpassungsmöglichkeiten, Unterstützung von Plugin-Authentifizierung sowie Beispiele.

## ⌨️ Lokale Entwicklung

Sie können GitHub Codespaces für die Online-Entwicklung nutzen:

[![][codespaces-shield]][codespaces-link]

Oder klonen Sie das Repository für die lokale Entwicklung:

```fish
$ git clone https://github.com/mDevsLabs/mAI.git
$ cd mAI
$ pnpm install
$ pnpm dev          # Full-Stack (Next.js + Vite SPA)
$ bun run dev:spa   # Nur SPA-Frontend (Port 9876)
```

> **Debug-Proxy**: Nach dem Ausführen von `dev:spa` gibt das Terminal eine Proxy-URL wie
> `https://app.mAI.com/_dangerous_local_dev_proxy?debug-host=http%3A%2F%2Flocalhost%3A9876` aus.
> Öffnen Sie diese, um lokal mit HMR gegen das Produktions-Backend zu entwickeln.

Wenn Sie weitere Einzelheiten erfahren möchten, lesen Sie bitte unser [📘 Entwicklungshandbuch][docs-dev-guide].
