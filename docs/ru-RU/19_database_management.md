# Управление базой данных 🗄️

**mAI** управляет хранением данных структурированным и высокопроизводительным образом, чтобы гарантировать быстрый офлайн-доступ к вашим разговорам и конфигурациям.

## Архитектура хранения

В зависимости от среды выполнения (десктопная или веб-версия), mAI использует различные механизмы хранения:
- **Десктопное приложение**: Использует легковесную локальную базу данных (SQLite), хранящуюся в пользовательском каталоге приложения. Это гарантирует отличную производительность чтения/записи для больших объемов истории.
- **Веб-приложение**: Опирается на внутреннюю базу данных браузера (IndexedDB) или внешние службы API, синхронизированные с базой данных PostgreSQL, если включен облачный режим.

## Миграции схем данных

mAI включает систему автоматической миграции схемы базы данных. При каждом обновлении приложения, если структура базы данных изменяется (например, добавляется новое поле для агентов или сообщений), миграции выполняются плавно при запуске без изменения существующих данных.

## Оптимизация производительности

Чтобы приложение оставалось быстрым:
- **Автоматическая очистка**: Временные сообщения или истекшие сессии могут периодически удаляться.
- **Индексация**: Сообщения разговоров индексируются для обеспечения почти мгновенного текстового поиска.
