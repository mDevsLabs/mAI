# データベース管理 🗄️

**mAI** は、会話や設定への迅速なオフラインアクセスを保証するために、構造化され、非常にパフォーマンスの高い方法でデータストレージを管理します。

## ストレージアーキテクチャ

実行環境（デスクトップまたはWeb）に応じて、mAIは異なるストレージエンジンを使用します：
- **デスクトップアプリケーション**：アプリケーションのユーザーディレクトリに保存されている軽量なローカルデータベース（SQLite）を使用します。大量の履歴に対して優れた読み書きパフォーマンスを保証します。
- **Webアプリケーション**：ブラウザの内部データベース（IndexedDB）を使用するか、クラウドモードが有効な場合はPostgreSQLデータベースと同期された外部のAPIサービスに依存します。

## スキーママイグレーション

mAIには、データベーススキーマの自動マイグレーションシステムが含まれています。アプリケーションが更新されるたびに、データベース構造が変更された場合（たとえば、エージェントやメッセージに新しいフィールドが追加された場合など）、起動時に既存のデータを変更することなくマイグレーションがシームレスに実行されます。

## パフォーマンスの最適化

アプリケーションを高速に維持するために：
- **自動クリーンアップ**：一時的なメッセージや期限切れのセッションは定期的にクリーンアップされます。
- **インデックス作成**：会話メッセージにはインデックスが作成され、ほぼ瞬時のテキスト検索が可能です。
