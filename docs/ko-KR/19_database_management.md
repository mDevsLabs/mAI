# 데이터베이스 관리 🗄️

**mAI**는 대화 기록 및 구성에 대한 신속한 오프라인 액세스를 보장하기 위해 구조화되고 우수한 성능의 데이터 저장소 관리 방식을 채택하고 있습니다.

## 스토리지 아키텍처

런타임 환경(데스크톱 또는 웹)에 따라 mAI는 서로 다른 스토리지 엔진을 활용합니다.
- **데스크톱 애플리케이션**: 애플리케이션의 사용자 디렉터리에 저장되는 가벼운 로컬 데이터베이스(SQLite)를 사용합니다. 이는 방대한 양의 대화 기록에 대해 뛰어난 읽기/쓰기 성능을 보장합니다.
- **웹 애플리케이션**: 브라우저 내부 데이터베이스(IndexedDB)에 의존하거나, 클라우드 모드가 활성화된 경우 PostgreSQL 데이터베이스와 동기화된 외부 API 서비스를 사용합니다.

## 스키마 마이그레이션

mAI에는 자동 데이터베이스 스키마 마이그레이션 시스템이 포함되어 있습니다. 애플리케이션 업데이트 시 데이터베이스 구조가 변경되는 경우(예: 에이전트 또는 메시지에 새 필드가 추가됨), 기존 데이터 변경 없이 시작 시 마이그레이션이 원활하게 실행됩니다.

## 성능 최적화

애플리케이션을 신속하게 유지하기 위해:
- **자동 정리**: 임시 메시지나 만료된 세션은 주기적으로 정리될 수 있습니다.
- **인덱싱**: 거의 즉각적인 텍스트 검색이 가능하도록 대화 메시지가 인덱싱됩니다.
