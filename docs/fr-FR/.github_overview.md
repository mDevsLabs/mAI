# 🛠️ Présentation du dossier .github

Ce dossier contient toute la configuration et la gouvernance du projet sur GitHub.

## 📂 Structure du Dossier

### 📁 ISSUE_TEMPLATE/

Contient les modèles de tickets pour standardiser les rapports de bugs, les demandes de fonctionnalités et la documentation.

### 📁 actions/

Définit des actions GitHub réutilisables pour automatiser des tâches comme la configuration de l'environnement ou le déploiement desktop.

### 📁 scripts/

Contient des scripts utilitaires pour la gestion du dépôt (ex: fermeture automatique des doublons).

### 📁 workflows/

Définit les pipelines de CI/CD (Continuous Integration / Continuous Deployment) pour le déploiement sur Vercel et la gestion des releases desktop.

## 📄 Fichiers Racines

- **CODEOWNERS** : Définit les responsables de chaque partie du code pour les revues de PR.
- **FUNDING.yml** : Configure les options de soutien financier pour le projet.
- **maintainers.txt** : Liste les mainteneurs officiels du projet.
- **PULL_REQUEST_TEMPLATE.md** : Modèle standard à remplir pour chaque demande de modification de code.
