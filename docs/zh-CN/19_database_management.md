# 数据库管理 🗄️

**mAI** 以结构化和高效的方式管理其数据存储，以保证对您的对话和配置进行快速、离线的访问。

## 存储架构

根据运行环境（桌面端或网页端）的不同，mAI 使用不同的存储引擎：
- **桌面端应用**：使用存储在应用用户目录中的轻量级本地数据库 (SQLite)。它为大量历史数据提供了出色的读/写性能。
- **网页端应用**：依赖于浏览器内部数据库 (IndexedDB)，或在启用云端模式时，与 PostgreSQL 数据库同步的外部 API 服务。

## 模式迁移 (Schema Migrations)

mAI 包含一个自动的数据库模式迁移系统。在每次应用更新时，如果数据库结构发生变化（例如，在智能体或消息中添加新字段），迁移将在启动时无缝运行，而不会更改您的现有数据。

## 性能优化

为了保持应用快速运行：
- **自动清理**：可以定期修剪临时消息或过期会话。
- **索引**：对对话消息进行索引以允许近乎瞬时的文本搜索。
