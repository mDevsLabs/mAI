# 資料庫管理 🗄️

**mAI** 以結構化且高效能的方式管理其資料儲存，以保證對您的對話和配置進行快速、離線的存取。

## 儲存架構

根據運行環境（桌面端或網頁端），mAI 使用不同的儲存引擎：
- **桌面應用程式**：使用儲存在應用程式使用者目錄中的輕量級本地資料庫（SQLite）。它保證了大型歷史記錄量具有極佳的讀寫效能。
- **網頁應用程式**：依賴瀏覽器的內部資料庫（IndexedDB）或在啟用雲端模式時與 PostgreSQL 資料庫同步的外部 API 服務。

## 綱要遷移 (Schema Migrations)

mAI 包含一個自動資料庫綱要遷移系統。在每次應用程式更新時，如果資料庫結構發生變化（例如，在智慧體或訊息中新增欄位），遷移程式將在啟動時無縫運行，而不會更改您現有的資料。

## 效能優化

為了保持應用程式的快速運行：
- **自動清理**：可以定期清理臨時訊息或過期的工作階段。
- **索引**：對聊天訊息進行索引，以允許近乎即時的文字搜尋。
